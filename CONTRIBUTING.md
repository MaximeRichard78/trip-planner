# Contributing

## Workflow Git

1. Toujours travailler sur une branche `feature/nom-feature`
2. Merger vers `develop` via Pull Request uniquement
3. Jamais de push direct sur `main`

## Format des commits — Conventional Commits

Types : `feat`, `fix`, `docs`, `test`, `ci`, `chore`, `refactor`

Exemples :
- `feat(routing): add stop ordering algorithm`
- `fix(ban-api): handle empty address response`
- `docs(readme): update launch procedure`
